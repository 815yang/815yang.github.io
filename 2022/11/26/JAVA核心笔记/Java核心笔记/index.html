1.5Jave的特性（11个）

简单、面向对象、健壮性、安全性、结构中立的、可移植性、高性能、解释执行、多线程、动态、平台无关。

详解:

1.简单：Java起初是为了控制家用电器分布式而设计的. Java摒弃了C++中容易引发错误的一些特性:如 执政 结构 操作重载 多重继承 . Java提供了丰富的类库

🔺🔺🔺为什么指针不安全? 主要还是程序员的操作不当 ①不知指针指向哪块内存区域 ②指针使用未初始化 ③指针使用后为释放 ④指针释放后 未NULL ⑤缓冲区读入 缓冲区溢出

野指针的利用： free double

2.面向对象: 完全面向对象 (封装 继承 多态)

3.健壮：Java是强类型语言，类型检查严格。且引入了垃圾内存回收(GC)机制，防止内存泄露问题。

4.安全：①删去指针 ②Java安全架构保证Java代码的安全性 ： Java安全架构保证代码不能随意访问我们本地计算机资源 如 删除文件 访问本地网络资源

5.结构中立：Java虚拟机 通过中间字节码(bytecodes)文件 执行程序。不担心计算机结构不同

6.可移植性：在备配了 Java解释器 和运行环境的计算机系统运行

Java 基本数据类型: byte - 8bit 二进制补码

short -16bit 二进制补码

int -32bit 二进制补码

long -64bit 二进制补码

float -32 bit IEEE 754 浮点数

double -64bit IEEE 754 浮点数

char -16bit Unicode 字符

7.高性能: Java可以在运行的时候将目标代码 翻译成机器指令， Java虚拟机中的 JIT(just in time 即时编译)实现

8.解释执行：Java编译器 将 java源代码编译成 字节码文件 。 通过解释器 解释字节码文件运行

9.多线程：Java的多线程库 在 Java的类库中

10.动态： 允许程序动态的 装入运行过程中 所需的类。

🔺🔺🔺动态怎么实现? Java编译器将符号引用信息保存在字节码中，传递给解释器，由解释器在完成动态链接类后，将符号引用信息转化为 数值偏移量。 这种符号信息查找 过程 是当一个新的名字出现一次，才进行

​ 总结，存储器中生成的对象 不是在编译过程中完成， 而是通过字节码， 在解释器中完成。

0.序言（运行时的数据区）

JVM中的堆啊、栈啊、方法区什么的，是Java虚拟机的内存结构，Java程序启动后，会初始化这些内存的数据。

Java 虚拟机在执行 Java 程序的过程中会把它管理的内存划分为若干个不同的数据区域。每个区域都有各自的作用。

JVM 内存结构分析，主要就是分析 JVM 运行时数据存储区域。

JVM 的运行时数据区主要包括：堆、栈、方法区、程序计数器等。

JVM 的优化问题主要在线程共享的数据区中：堆、方法区。

运行时数据区

1.程序计数器：记录程序执行到哪一个指令

2.本地方法栈：与虚拟机栈功能相似，不过虚拟机栈为java方法服务，本地方法栈为Native方法服务

3.方法区：存放字节码，常量 ，静态变量,是一个共享的区域

4.虚拟机栈：执行方法其实就是栈帧入栈，出栈的过程，因为是跟方法有关的，而且局部变量常常在方法中，所以栈存储的是局部变量

5.堆：存放引用类型，成员变量

注意：(1).我们接触最多的是堆和栈(虚拟机栈)，这里一定要注意这两个的区别。

(2).栈主要是存储方法帧的，每执行一个方法的时候，就会为该方法创建一个栈帧，使其入栈。然而局部变量一般随着方法生成而存在，所以局部变量存储在栈中。

(3).堆一般用来存放引用型对象，以及对应的成员变量，当对象创建时，会在堆中开辟一块地址，并给这个对象的成员变量使用。
